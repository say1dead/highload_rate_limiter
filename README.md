# Rate Limiter

Микросервисная система для централизованного управления пользователями и ограничения количества запросов (Rate Limiting). Проект построен на **ASP.NET Core**, использует **gRPC** для взаимодействия между сервисами, событийную архитектуру на **Apache Kafka** и распределённое хранение данных.

## Основные возможности

* Управление пользователями и их лимитами запросов.
* Проверка превышения лимитов в режиме реального времени.
* Автоматическая блокировка пользователей при превышении допустимого количества запросов.
* Синхронизация данных между сервисами через Apache Kafka.
* Кэширование лимитов в памяти с автоматическим обновлением через MongoDB Change Streams.
* Хранение информации о превышениях лимитов в Redis с использованием TTL.

## Архитектура

Система состоит из нескольких микросервисов:

### User Service

* Управление пользователями.
* Хранение данных в PostgreSQL.
* gRPC API для взаимодействия с другими сервисами.

### Limit Write Service

* Создание и изменение лимитов.
* Сохранение лимитов в MongoDB.
* Публикация событий об изменениях в Apache Kafka.

### Limit Read Service

* Предоставление актуальных лимитов.
* Кэширование данных в памяти.
* Автоматическое обновление кэша через MongoDB Change Streams.

### Rate Limiter Service

* Проверка запросов пользователей.
* Подсчёт количества запросов.
* Использование Redis (TTL) для хранения счётчиков.
* Блокировка пользователей при превышении лимита.

## Используемые технологии

* ASP.NET Core
* gRPC
* MongoDB
* PostgreSQL
* Redis
* Apache Kafka
* Docker & Docker Compose

## Особенности реализации

* Асинхронная обработка запросов (`async/await`).
* Dependency Injection.
* Конфигурация через `IOptions`.
* gRPC Interceptors для:
  * валидации входящих запросов;
  * проверки лимитов;
  * автоматической блокировки пользователей.
* In-Memory Cache для ускорения чтения лимитов.
* MongoDB Change Streams для синхронизации кэша без постоянного опроса базы данных.
* Событийное взаимодействие сервисов через Apache Kafka.
